When you register normally it is treated like you want to setup a separate company for this app because this app support multiple company

People of different company strictly MUST NOT be able to communicate with each other so company_id is critical for separation

Core Rules (Do Not Violate)
- Every user belongs to exactly one company
- company_id is required on all user-owned records
- Users from different companies must never see or interact with each other
- There is no cross-company communication, data sharing, or visibility

When you register you become the admin, admin have previlege of inputting new user of the same company, so anyone that want to be part of the company must be registered by admin and can't register from the front
